# Phoenix7 v3N1 — Living World: цикл день-ночь и распорядок жителей

## Цель

Сделать мир тракта Порт Рейчел — Форт Заря живым во времени, оставаясь на стороне
«Морровинда» (мир и атмосфера). Проход не трогает героя, тело, руки, оружие и
ощущение боя — это параллельная работа Codex (`v3M2A`). Никакой смены координат,
квест-системы, save-сериализации и внешних ассетов.

Ветка делалась в отдельном git worktree от `main`, чтобы не задеть рабочее дерево
Codex.

## Запуск

```text
START_GAME_V3L_WINDOWS.bat
http://localhost:8000/v3l.html
```

Совместимый вход: `http://localhost:8000/v3k.html`. После правок исходных модулей —
жёсткий перезапуск `Ctrl+F5`.

Клавиша `T` — переключение скорости времени (×1 → ×30 → ×180 → ×900) для проверки
всего цикла без ожидания. Часы видны в правом верхнем углу.

## Что добавлено

### Блок 1 — цикл день-ночь

Новые файлы:

- `src/game/v3/world/dayNight.js` — общий `worldClock` (мутабельный, читается
  расписанием NPC без связности) и класс `TimeOfDay` (фаза 0…1, длина суток
  ~12 реальных минут на ×1, четыре сегмента: рассвет/день/сумерки/ночь).
- `src/game/v3/core/engineDayNightExtensions.js` — пост-проход поверх
  `VisualAtmosphereSystem`, луна, звёздное небо, HUD-часы, клавиша времени,
  диагностика.

Каждый кадр (после биомной атмосферы) система берёт **стабильную базу текущего
биома** (`visualAtmosphere.profile`) и накладывает кривую суток — без обратной
связи с уже задемпленными значениями:

- **Солнце** — позиция по дуге (восток→зенит→запад, ниже горизонта ночью),
  интенсивность гаснет к ночи, цвет теплеет в сумерках; та же `sunDirection`
  идёт в шейдер неба (диск солнца).
- **Небо** — зенит/горизонт/пыль гнутся к ночной палитре по `nightFactor`,
  горизонт теплеет по `twilight`.
- **Туман** — плотнее и холоднее ночью (×до 1.55), тёплый оттенок в сумерках;
  фон сцены следует за цветом тумана.
- **Ambient/Hemisphere** — тусклее ночью (ambient до 0.30 базовой, hemi до 0.32).
- **Луна** — отдельный `DirectionalLight` напротив солнца, появляется ночью.
  На **Low** не создаётся (как и прочие realtime-источники).
- **Звёзды** — один `Points` (~620 точек, на Low ~280), проявляются по
  `nightFactor`, рецентрируются на игрока, чтобы не было параллакса.

### Блок 2 — распорядок жителей

`src/game/v3/world/life.js`: у жителей поселений (с `settlementId`) появился
`homeAnchor` (= точка спавна) и `scheduleState`. Днём — рабочие маршруты как
раньше; ночью (`worldClock.nightFactor > 0.55`) житель смещается к дому
сниженным шагом и стоит у дома. Караваны и общие бродячие NPC расписанию не
подчиняются и продолжают движение. Панель разговора показывает строку
«Распорядок: …».

### Блок 3 — события по времени суток

`life.js` при смене сегмента суток добавляет короткую строку в журнал
(`eventLog`): рассвет — выход соляного каравана, день — возвращение общин к
работе, сумерки — смена дозоров, ночь — костры и расход по домам. Без боя,
экономики и квестов.

## Архитектура и совместимость

- Подключение — одна строка `installDayNightExtensions` в `main.js`, после
  `installAtmosphereExtensions` (пост-проход обязан идти последним). `engine.js`
  не правится.
- HUD-часы создаются из JS (`createElement`), HTML-входы `v3l.html`/`v3k.html`
  не трогаются.
- Расписание NPC читает `worldClock` через импорт `dayNight.js` — без изменения
  сигнатур движка.

Файлы прохода:

| Файл | Тип |
|---|---|
| `src/game/v3/world/dayNight.js` | новый — TimeOfDay + worldClock |
| `src/game/v3/core/engineDayNightExtensions.js` | новый — визуальный пост-проход, луна/звёзды, часы, клавиша T, диагностика |
| `src/game/v3/world/life.js` | правка — homeAnchor, распорядок, события по времени |
| `src/game/v3/main.js` | правка — один импорт + install |

## Диагностика

```js
PHX_V3_ENGINE.getDayNightDiagnostics()
```

Возвращает фазу, часы, сегмент, `sunAltitude`, `dayFactor`, `nightFactor`,
`twilight`, `timeScale`, текущую интенсивность солнца/ambient, плотность тумана,
интенсивность луны и прозрачность звёзд.

## Производительность

Замеры — детерминированным форс-рендером из гущи кластера (`rig (12,150)`),
по всей сцене (ветка = `main` + v3N1; абсолютная база выше прежних v3M1.1-замеров,
потому что в `main` уже влита нарративная интеграция и масштабируемая карта).

| Пресет | Draw calls (полдень) | Draw calls (полночь) | Triangles | Луна |
|---|---:|---:|---:|:--:|
| Low | 514 | 521 | ~64 000 | выключена |
| Medium | 552 | 553 | ~91 000 | есть |
| High | 593 | 600 | ~118 000 | есть |

**Собственная стоимость прохода проверена прямой изоляцией (Medium, полночь):
+1 draw call** — звёздное поле (один `Points`). Луна — источник света, 0 draw
calls; днём звёзды скрыты (`visible=false`), стоимость 0. На Low луна не
создаётся. Никаких per-frame аллокаций в пост-проходе (временные `Color`/`Vector`
переиспользуются).

## Ограничения и дальнейшее

- Ночь — это освещение, туман, луна и звёзды; погодных систем и облаков нет.
- Распорядок — две фазы (работа/отдых) с якорем дома; полноценных суточных
  расписаний с точками «обед/торговля» пока нет.
- Время не сериализуется в save (как и весь мир сейчас).
- Возможное продолжение: фонари/костры, зажигающиеся ночью; враги активнее ночью;
  сон/ожидание игрока с проматыванием времени.

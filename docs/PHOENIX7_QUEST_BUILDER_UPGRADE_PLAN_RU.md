# Сборщик квестовых драфтов — план апдейта и улучшения интерфейса (для Codex/GPT)

Файл: `quest_draft_builder.html` (корень репо). Один автономный HTML, vanilla JS,
без зависимостей, работает оффлайн, состояние в `localStorage`. **Эти принципы
сохранять:** один файл, без сборки, без внешних библиотек (Mermaid/иконки —
инлайнить или генерировать текстом), офлайн, ничего не шлёт в сеть.

Текущее: форма (шапка / NPC / завязка / ветки по группам / глобально) → две
вкладки вывода (Markdown-черновик и engine-aware промпт) → save/load + JSON
import/export + сид-пример. Структура кода: `state={meta,npcs,branches}`,
функции `renderNpcs/renderBranches/buildMarkdown/buildPrompt/sync`.

Делать по фазам, каждая — отдельный коммит. После правок открыть файл в браузере,
проверить консоль (0 ошибок), прогнать сид-пример и обе вкладки.

---

## Фаза 1 — UX-победы (быстро, высокий эффект)

1. **Сворачиваемые секции** карточек (`<details>`/кнопка-стрелка), запоминать
   открытость в localStorage. Длинная форма перестаёт пугать.
2. **Перетаскивание веток** (reorder) — drag-handle на `.branch` (HTML5
   draggable или стрелки ↑/↓ как минимум). Порядок = порядок в выводе/группах.
3. **Дублирование ветки** (кнопка «копировать ветку») — частый кейс «такой же, но
   для другой расы».
4. **Лайв-валидация (lint-панель)** над выводом: подсветить пустой `id`/название;
   дублирующиеся диапазоны стадий между ветками; ветки без «исхода/награды»;
   флаги, упомянутые в эффектах, но не перечисленные в «Глобально». Не блокировать
   — только предупреждать (жёлтые чипы).
5. **Несколько целей промпта** — выпадашка/кнопки: «Промпт: дизайн-разбор», «Промпт:
   готовый код+слоты», «Промпт: только dialogueData-блок». Разные хвосты-инструкции
   к одному драфту (уже есть `buildPrompt` — параметризовать `mode`).
6. **Mermaid-схема потока** в выводе (третья вкладка «Схема»): генерировать текст
   ```mermaid flowchart``` из стадий/веток (узлы = стадии, рёбра = переходы по
   route). Просто текст в `<pre>` — рендер не нужен, его сделает нейронка/доки.
7. **Счётчики и QoL:** число веток/NPC в шапке; кнопка «копировать секцию»; тёмная/
   светлая тема-тумблер; горячая клавиша Ctrl+S = «Сохранить как».

## Фаза 2 — закрыть петлю к коду (главная ценность)

Цель: на выходе не только проза, но **готовый к вставке блок `dialogueData.js` +
карта стадий `questData.js`**, чтобы нейронка/автор сразу вставляли в игру.

1. **Авторинг реплик внутри ветки.** В каждую ветку добавить под-список «реплики»:
   - поле NPC (id из списка действующих лиц), стадия-условие (`quest:{id:'>=N'}`),
   - текст реплики,
   - список «выборов» (label + условие + эффекты + `say`/`goto`).
   Данные: `branch.replies=[{npc,when,text,choices:[{label,when,effects,say,goto}]}]`.
2. **Структурные пикеры условий/эффектов** (вместо свободного текста):
   - Условие `when`: чипы-конструктор — race(мультивыбор), background, flag/notFlag,
     quest `id` + оператор + N, skillAbove, hasItem, any/all. Сериализовать в объект.
   - Эффект: тип из списка (`setFlag/quest/questPatch/questItem/reward/reputation/
     credits/journal`) + поля под тип. Массив эффектов на выбор.
   Это гарантирует валидный синтаксис движка.
3. **Кодоген.** Кнопка «Сгенерировать dialogueData-блок» → собрать из `npcs`+`replies`
   валидный JS-объект `DIALOGUE['npc-id']={name,greeting:[…],topics:{…}}` с
   условиями/эффектами; и «Сгенерировать questData» → `{stages:{N:'…'}}` из стадий
   веток. Выводить в отдельную вкладку «Код».
4. **Слоты-заглушки.** Где текста нет — выводить `text:'АВТОР: …'`, чтобы было
   очевидно, что дописать.

## Фаза 3 — продвинутое

1. **Библиотека шаблонов квестов:** «принеси/убей/эскорт/расследование/выбор
   фракции» — пресеты `state`, как сид-пример, в выпадашке.
2. **Синхронизация с движком (опц.):** если файл открыт и в той же вкладке есть
   `window.PHX_V3_ENGINE` — автодополнять id NPC (`livingWorld.agents`), список
   существующих флагов (`worldState.data.flags`), занятые стадии. Если движка нет —
   просто скрыть.
3. **История ревизий драфта** (localStorage стек, кнопка «откатить»).
4. **Планировщик экономики/репутации:** таблица наград (кредиты/предметы) и
   изменений репутации по исходам — наглядно балансировать.
5. **Экспорт пакета:** «Скачать всё» → zip-подобный единый .md (драфт + промпт +
   код + схема) или несколько файлов через несколько `download`.

## Технические заметки

- Всё — в одном `<script>` в `quest_draft_builder.html`. Расширять `state`-модель
  обратносовместимо (`normalize()` дозаполняет новые поля).
- Никаких внешних CDN. Иконки — юникод. Схема — текст Mermaid в `<pre>`.
- Сохранять автосейв `phoenix7_quest_drafts_current` и именованные
  `phoenix7_quest_drafts`. Не ломать существующие сохранения (миграция в
  `normalize`).
- Каждую фазу проверять: открыть файл, консоль 0 ошибок, сид-пример заполняется,
  все вкладки вывода непустые, save→load восстанавливает, import JSON работает.

## Приоритет

Сначала **Фаза 1** (дешёвая, делает инструмент приятным), затем **Фаза 2** (даёт
готовый код на выходе — это окупает всё), Фаза 3 — по желанию. Каждая фича
самодостаточна; можно брать по одной.

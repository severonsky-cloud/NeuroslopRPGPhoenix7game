# Phoenix7 Version 3 — план перехода

## Зачем нужна версия 3

Версия 2.5 доказала, что направление работает:

- игра запускается локально без Python через `START_GAME_WINDOWS.bat`;
- есть 3D-камера от первого лица;
- есть руки и оружие;
- есть первый квестовый маршрут;
- есть живые NPC;
- есть несколько биомов;
- есть рельеф;
- есть боёвка, лут и разговоры.

Но текущий билд уже достиг лимита прототипа:

- мир тормозит из-за слишком большого количества отдельных Three.js-объектов;
- все системы всё ещё лежат в одном огромном `src/game/main.js`;
- графика работает как blockout, а не как художественная сцена;
- NPC живые только базово;
- квестовая структура пока линейная и маленькая;
- монстры уже разные по силуэту, но ещё не полноценные модели;
- нет нормального world streaming / chunk system.

**Версия 3 нужна, чтобы не ломать достижения, а превратить их в нормальный фундамент игры.**

---

## Главный принцип v3

```text
Не добавлять бесконечно кубы в один файл.
Нужно перейти к архитектуре игры.
```

Версия 3 должна быть не просто новым красивым `main.js`, а системой:

```text
game.html
src/game/main.js
src/game/core/engine.js
src/game/core/input.js

src/game/world/roads.js
src/game/world/props.js
src/game/entities/player.js
src/game/entities/npc.js
src/game/entities/monster.js
src/game/combat/weapons.js
src/game/combat/melee.js
src/game/combat/projectiles.js
src/game/quests/act1.js

```

## Главные цели v3

### 1. Производительность

Текущий билд тормозит из-за количества объектов и материалов. Нужно:

- заменить тысячи отдельных камней/деревьев на `InstancedMesh`;
- сделать чанки мира;
- рисовать только ближайшие чанки;
- добавить LOD;
- ограничить pixel ratio;
- отключать дальние подписи;
- сделать quality presets: Low / Medium / High;
- уменьшить число realtime lights;
- заменить часть PointLight на emissive materials;
- не создавать случайные объекты каждый раз хаотично, а хранить seed.

### 2. Графика, которая цепляет

Текущая графика — рабочий blockout. Нужно сделать художественный стиль:

```text
Morrowind exploration
+ Medal of Honor weapon feel
+ red clay Africa / Gulf coast
+ strange cosmic frontier
+ no glossy modern look
```

В v3 нужно:

- цветовые LUT-палитры по биомам;
- утренний/вечерний свет;
- туман слоями;
- больше вертикальных силуэтов;
- читаемые дороги;
- более выразительные руины;
- настоящие landmark-объекты;
- атмосферные sky gradients;
- water/shore shader-lite;
- glass forest glow;
- ice shelf cold haze;
- tsarbor forest green-black fog.

### 3. Живой мир

Нужно перейти от `NPC стоит/ходит` к системе жизни.

NPC должны иметь:

- имя;
- фракцию;
- маршрут;
- дневной цикл;
- роль;
- слухи;
- торговлю/задание/комментарий;
- реакцию на игрока;
- реакцию на бой рядом;
- принадлежность к зоне.

Примеры:

```text
портовый патруль ходит от доков к таможне
караван идёт от соляных доков к Форту Заря
смотритель шельфа ходит вдоль ледяных плит
стеклянный монах ходит вокруг чёрных обелисков
разведчик царборцев патрулирует лесную тропу
```

### 4. Полноценная карта мира

Версия 3 должна сделать карту не просто большим прямоугольником, а миром с регионами:

```text
Океан и береговой старт
Порт Рейчел
Соляные доки
Мангровые болота
Красная дорога
Старая батарея
Форт Заря
Красный Узел
Край Мёртвой Саванны
Лес царборцев
Вотчина чёрных элементалей / Стеклянный лес
Ледяной шельф
Дальняя сухая саванна
```

Карта должна иметь:

- дороги;
- тропы;
- высоты;
- низины;
- безопасные места;
- опасные зоны;
- landmark-силуэты;
- переходы между биомами.

### 5. Боевая система

Сейчас бой работает, но должен стать приятным.

Нужно отдельно сделать:

- melee sweep;
- hitboxes;
- parry/block;
- stagger;
- enemy attack telegraph;
- phase punch cone;
- weapon recoil curves;
- weapon sway curves;
- impact particles;
- simple damage types:
  - kinetic;
  - blade;
  - phase;
  - glass;
  - ice;
  - black elemental.

Оружие:

```text
1 — кулаки
2 — фазовый удар рукой
3 — бастард
4 — шпага
5 — Кольт 1917
6 — M1 Гаранд
7 — Брен
```

Оружие остаётся художественным игровым элементом мира, а не реалистичной инструкцией.

### 6. Монстры

Текущие монстры должны стать archetype-моделями.

Нужные типы:

```text
road brute — дорожный рогатый мутант
swamp lurker — болотная многоногая тварь
bone ghoul — сухая костяная падаль
glass predator — стеклянный хищник
ice carrion — ледяной падальщик
phase echo — фазовое эхо
black elemental — чёрный элементаль стекла
```

Для v3:

- каждый монстр получает свой силуэт;
- свой цвет;
- свой тип движения;
- свой звук/визуальный след;
- свой лут;
- свою роль в биоме.

### 7. Квестовый луп Act 1

Текущий первый акт нужно превратить в полноценный цикл:

```text
Берег
→ Рина / Таможня
→ Дорожный навес
→ Оран / регистрация
→ Герда / доска подготовки
→ 4 подготовки:
   1. дорога
   2. рынок
   3. фракции
   4. контрабанда
→ Сава
→ переход к Мёртвой Саванне
```

В v3 нужно добавить:

- реальные objective markers;
- журнал с текущим шагом;
- несколько способов выполнить подготовку;
- маленькие побочные события;
- награду от Герды;
- unlock следующего региона.

- добавить chunk manager;
- заменить деревья/камни/кристаллы на `InstancedMesh`;
- quality preset;
- меньше realtime lights;
- меньше DOM labels одновременно;

- новая цветовая палитра;

## Acceptance Criteria для v3

Версия 3 считается успешной, если:

```text
1. Игра запускается через START_GAME_WINDOWS.bat.
2. FPS заметно лучше, чем в 2.5H/2.5I.
3. Мир выглядит красивее, не как плоский blockout.
4. Есть минимум 6 биомов.
5. Есть живые NPC на маршрутах.
6. Есть caravan/patrol/roaming encounters.
7. Есть полноценный Act 1 loop.
8. Мечи, фаза и огнестрел ощущаются разными.
9. Монстры отличаются силуэтом.
10. Стеклянный лес ясно ощущается вотчиной чёрных элементалей.
```

```text
v3.0A Performance Foundation
```

То есть:

- создать новую структуру `src/game/v3/`;
- перенести рабочие идеи из 2.5;
- сделать оптимизированный world/chunk prototype;
- оставить 2.5 как playable backup.

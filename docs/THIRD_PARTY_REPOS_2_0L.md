# Phoenix7 2.0L — third-party repositories review

Цель: понять, какие внешние open-source репозитории можно использовать без превращения проекта в лицензионный и технический хаос.

## Главный принцип

Можно использовать сторонние репозитории, но только если:

1. У проекта понятная лицензия.
2. Лицензия совместима с нашим браузерным билдом.
3. Библиотека не ломает локальный запуск.
4. Есть CDN/ESM вариант или простой способ положить файл локально.
5. Есть fallback, если библиотека не загрузилась.

## Разрешённые типы лицензий

Лучшие для нас:

- MIT
- BSD-2-Clause
- BSD-3-Clause
- ISC
- Apache-2.0

Осторожно:

- GPL
- AGPL
- CC-BY-SA для ассетов
- любые неизвестные кастомные лицензии

## Кандидаты

### already in use

- `three` — базовый 3D-рендер.
- Three.js examples / GLTFLoader — загрузка GLB.

### useful next

- `cannon-es` — физика и коллизии, если текущих простых коллизий станет мало.
- `lil-gui` — debug/settings panel для времени суток, телепорта, света.
- `stats.js` — FPS/debug overlay.
- `simplex-noise` или похожая маленькая noise-библиотека — terrain/облака/миры.
- `idb-keyval` — нормальные сохранения, если localStorage станет мал.

## Что не делать

- Не тащить огромный движок ради одной функции.
- Не подключать 10 библиотек сразу.
- Не использовать ассеты без явной лицензии.
- Не смешивать GPL-код с нашим HTML без понимания последствий.
- Не вставлять чужой код без сохранения license notice.

## Вывод

Да, сторонние репозитории использовать можно. Но только как контролируемые зависимости: маленькими слоями, с лицензиями, fallback и документацией.

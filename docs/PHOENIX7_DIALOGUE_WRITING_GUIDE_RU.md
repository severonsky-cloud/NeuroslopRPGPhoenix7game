# Phoenix7 — как писать диалоги (гайд для автора)

Файл, где ты пишешь: `src/game/v3/dialogue/dialogueData.js`.
Движок (`dialogueEngine.js`) только читает — его трогать не нужно.

## Как это работает (Морровинд-логика)

- У NPC есть **приветствие** и **темы**. Игрок кликает тему → NPC выдаёт реплику.
- В тексте реплики **известное слово-тема становится гиперссылкой** (по `TOPIC_LABELS`).
- На каждую тему берётся **первая подходящая** реплика из списка — поэтому
  специфичные реплики (по расе/флагу) пиши **выше**, общую — внизу.
- Реплика может **открыть новые темы** (`adds`) и дать **варианты ответа** игроку
  (`choices`), которые двигают квест/ставят флаги.

## 7 правил качественного диалога

1. **Голос, а не справка.** NPC говорит как человек со своим интересом, а не как
   подсказка. Плохо: «Иди в Ланг-До, проверь глину». Хорошо: «Артель прячет глину
   от описи. Докажешь — поделюсь штрафом. Покроешь их — твой риск.»
2. **Коротко.** 1–3 фразы на реплику. Длинные монологи режь на темы.
3. **Тема = крючок.** Каждая `adds`-тема должна звать кликнуть: упомяни её прямо
   в тексте словом из `TOPIC_LABELS`, чтобы появилась ссылка.
4. **Условия — это характер, а не текст-дубль.** Реакция на расу/флаг должна
   менять *отношение*, а не просто переставлять слова. Красному — подозрение;
   «был у фазовых магов» — раздражение чиновника.
5. **Выбор должен стоить.** В `choices` давай развилку с последствием
   (флаг/квест), а не просто «ок». Хотя бы один выбор — расовый/предысторийный.
6. **Не лги игроку молча.** Если выбор ставит флаг вражды — пусть NPC это покажет
   репликой (`say`), чтобы игрок понял цену.
7. **Оставляй воздух для повторного захода.** Первая подходящая реплика по
   `quest:'>=N'` пусть меняется по стадии — мир реагирует на прогресс.

## Шаблон (копируй и переписывай)

```js
'npc-id': {
  name: 'Имя в окне',
  start: ['topicA'],            // темы, видимые сразу
  greeting: [
    { when:{ quest:{ my_quest:'>=2' } }, text:'Реакция на прогресс.' },
    { when:{ race:'red' },               text:'Реакция на расу.' },
    { text:'Обычное приветствие.' },     // общий вариант — внизу
  ],
  topics: {
    topicA: [
      { when:{ flag:'was_in_phase_guild' }, text:'Уникальная реплика, …фазовые маги…', adds:['topicB'] },
      { text:'Обычная реплика, …красная глина…', adds:['topicB'] },
    ],
    topicB: [
      { text:'Развилка.',
        choices:[
          { label:'(Взяться)', effects:[{type:'quest', id:'my_quest', stage:1}, {type:'journal', text:'…'}], goto:'__close' },
          { label:'(Красный) «Своих не сдам.»', when:{race:'red'}, effects:[{type:'setFlag', flag:'sided_red'}], say:'NPC показывает цену.' },
          { label:'Отойти', goto:'__close' },
        ] },
    ],
  },
}
```

## Справочник условий (`when`) и эффектов

**Условия:** `race`/`notRace`, `background`, `gender`, `flag`/`notFlag`,
`quest:{ id:'>=2' }`, `skillAbove:{ phase:20 }`, `hasItem:'permit'`, `chance:0.5`.
Несколько полей = И (все должны совпасть).

**Эффекты:** `{type:'quest', id, stage}` · `{type:'setFlag', flag, value}` ·
`{type:'credits', amount, once:true}` · `{type:'journal', text}`.
`once:true` — срабатывает один раз навсегда.

**Гиперссылки:** добавь слово в `TOPIC_LABELS` (`id -> 'слово'`), и оно станет
ссылкой в любой реплике, где тема уже открыта.
